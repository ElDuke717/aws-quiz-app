AWS IAM Study Guide

Key Concepts and Definitions

  • AWS IAM (Identity and Access Management): A service that helps you securely control access to AWS services and resources for your users.
  • Users: Represents an individual person or service that interacts with AWS.
  • Groups: A collection of IAM users that can be managed as a single entity.
  • Roles: An IAM identity with specific permissions that can be assumed by users or services.
  • Policies: Documents that define permissions and are attached to users, groups, or roles.
  • Multi-Factor Authentication (MFA): An extra layer of security that requires not only a username and password but also something that only the user has (like a security token).

Mnemonics or Memory Aids

  • U.G.R.P: Remember "User Groups Roles Policies" as the core components of IAM.
  • R.U.L.E: "Roles Users List Permissions" to remember how access is structured in IAM.

Outlines of Processes

Creating an IAM User

  1. Sign in to the AWS Management Console.
  2. Navigate to IAM service.
  3. Select "Users" from the sidebar.
  4. Click on "Add user".
  5. Provide a username and select access type (programmatic and/or AWS Management Console).
  6. Set permissions (attach existing policies, copy permissions from another user, or create a new policy).
  7. Review and create the user.

Potential Points of Confusion

  • Understanding the difference between IAM Users and IAM Roles, particularly in regards to long-term vs. temporary access.
  • Confusing IAM Groups with IAM Users; Groups are collections of Users, not identities themselves.
  • The concept of permissions can be complex; understanding how policies work and how they are attached is crucial.
  • Multi-Factor Authentication (MFA) requirements may vary based on organizational security policies.
